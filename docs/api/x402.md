# x402 Payment Rails

The x402 standard (`HTTP 402 Payment Required`) is a protocol for direct, account-free commerce over HTTP. aeqi exposes two critical operations via x402:

1. **Inference API** — Pay per request for LLM completions
2. **Company Genesis** — Pay $19 USDC to create a Company immediately

No API keys, no account creation, no waiting. Just sign a cryptographic authorization and execute.

## Overview

x402 gates select endpoints with payment requirements. When you call a gated endpoint:

1. If you omit the `X-Payment` header, the server returns `402 Payment Required` with the payment terms.
2. You construct an [EIP-3009](https://eips.ethereum.org/EIPS/eip-3009) (USDC-specific) signed authorization, base64-encode it, and send it as the `X-Payment` header.
3. The server verifies the signature, settles the payment on-chain via a facilitator, and executes your request.

**Asset**: USDC on Base mainnet (6 decimal places).  
**No custody**: You sign directly with your private key; aeqi never holds your keys.  
**Verifiable**: All payments settle as on-chain transactions in the Base Ethereum block explorer.

## Endpoints

### Company Genesis

**`POST /api/companies/create`** (x402-gated)

Create a Company without an account. The payer address (extracted from the EIP-3009 signature) becomes the owner.

**Payment Required**: $19 USDC (19,000,000 atomic units).

**Request** (POST body, JSON):

```json
{
  "name": "Acme Corp",
  "blueprint": "aeqi",
  "owner_address": "0x...",
  "roles": [
    {
      "name": "CEO",
      "owner": "0x...",
      "vesting_months": 36
    }
  ]
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | string | Yes | Company display name, 1–64 characters. |
| `blueprint` | string | No | Template slug (defaults to `"aeqi"`). Must exist on-chain. |
| `owner_address` | string | No | Ethereum address (0x-prefixed). Defaults to the x402 payer address if omitted. |
| `roles` | array | No | Role definitions (passed to the blueprint; implementation deferred). |

**Response** (200 OK, JSON):

```json
{
  "entity_id": "ent_a1b2c3d4",
  "trust_address": "0x... | pending",
  "runtime_url": "https://ent-a1b2c3d4.aeqi.ai",
  "owner": "0x...",
  "blueprint": "aeqi",
  "payment_tx": "0x..."
}
```

| Field | Type | Notes |
|-------|------|-------|
| `entity_id` | string | Immutable, globally unique Company identifier. |
| `trust_address` | string | Smart contract address (TRUST) representing the Company on-chain. May be `"pending"` if DAO provisioning is still underway. |
| `runtime_url` | string | HTTP endpoint for communicating with this Company's runtime agent. |
| `owner` | string | Ethereum address of the owner (either the provided address or the x402 payer). |
| `blueprint` | string | Template slug used for initialization. |
| `payment_tx` | string | Confirmed Base mainnet transaction hash. |

**Timeline**:
- **Synchronous**: Company entity created, sandbox runtime spawned.
- **Asynchronous**: DAO provisioning (on-chain TRUST contract creation). Status polled via `GET /api/companies/{entity_id}`.

## Payment Flow

### Step 1: Request the Endpoint Without Payment

```bash
curl -X POST https://app.aeqi.ai/api/companies/create \
  -H "Content-Type: application/json" \
  -d '{"name": "My Company"}'
```

**Response (402 Payment Required)**:

```json
{
  "error": "payment_required",
  "paymentRequired": {
    "amount": "19000000",
    "asset": "0x833589fcd6edb6e08f4c7c32d4f71b54cf85d21d",
    "recipient": "0x...",
    "chainId": 8453,
    "network": "base",
    "description": "Create a Company on aeqi ($19 USDC one-time)",
    "scheme": "exact",
    "maxTimeoutSeconds": 300,
    "mimeType": "application/json",
    "resource": "/api/companies/create",
    "outputSchema": { ... }
  }
}
```

Extract the `paymentRequired` object — it contains all the cryptographic parameters you need.

### Step 2: Sign an EIP-3009 Authorization

Use the `paymentRequired` payload to construct and sign an EIP-3009 permit. This is a single Ethereum transaction that combines a USDC transfer and a signature proof.

**TypeScript Example** (using [viem](https://viem.sh)):

```typescript
import {
  createPublicClient,
  createWalletClient,
  http,
  encodeFunctionData,
} from "viem";
import { privateKeyToAccount } from "viem/accounts";
import { base } from "viem/chains";

const USDC_ADDRESS = "0x833589fcd6edb6e08f4c7c32d4f71b54cf85d21d";
const BASE_RPC = "https://mainnet.base.org";

// Your private key (test key only — never commit real keys).
const account = privateKeyToAccount("0x...");

const publicClient = createPublicClient({
  chain: base,
  transport: http(BASE_RPC),
});

const walletClient = createWalletClient({
  account,
  chain: base,
  transport: http(BASE_RPC),
});

// From the 402 response:
const paymentRequired = {
  amount: "19000000",
  asset: "0x833589fcd6edb6e08f4c7c32d4f71b54cf85d21d",
  recipient: "0x...",
  chainId: 8453,
  // ... more fields
};

// Get the current nonce for EIP-3009 (prevents replays).
const nonce = BigInt(Math.floor(Date.now() / 1000));

// Construct the USDC permit (EIP-3009 `transferWithAuthorization`).
// The signature proves you authorized this transfer without needing explicit approval.
const signature = await walletClient.signTypedData({
  account,
  domain: {
    name: "USD Coin",
    version: "2",
    chainId: 8453,
    verifyingContract: USDC_ADDRESS,
  },
  types: {
    TransferWithAuthorization: [
      { name: "from", type: "address" },
      { name: "to", type: "address" },
      { name: "value", type: "uint256" },
      { name: "validAfter", type: "uint256" },
      { name: "validBefore", type: "uint256" },
      { name: "nonce", type: "bytes32" },
    ],
  },
  primaryType: "TransferWithAuthorization",
  message: {
    from: account.address,
    to: paymentRequired.recipient,
    value: BigInt(paymentRequired.amount),
    validAfter: 0n,
    validBefore: BigInt(Math.floor(Date.now() / 1000)) + 300n, // 5 minute window
    nonce: `0x${nonce.toString(16).padStart(64, "0")}`,
  },
});

// Encode the payment authorization as JSON.
const paymentAuthorization = {
  scheme: "eip3009",
  signature: signature,
  from: account.address,
  to: paymentRequired.recipient,
  amount: paymentRequired.amount,
  nonce: `0x${nonce.toString(16).padStart(64, "0")}`,
  chainId: 8453,
};

// Base64-encode for the header.
const encoded = Buffer.from(JSON.stringify(paymentAuthorization)).toString(
  "base64"
);
const paymentHeader = `x402 ${encoded}`;

console.log("Payment header ready:", paymentHeader);
```

### Step 3: Retry With the Payment Header

Add the `X-Payment` header to your request:

```bash
curl -X POST https://app.aeqi.ai/api/companies/create \
  -H "Content-Type: application/json" \
  -H "X-Payment: x402 <base64-encoded-authorization>" \
  -d '{"name": "My Company"}'
```

### Step 4: Server Verifies and Settles

The server:

1. Decodes the `X-Payment` header.
2. Calls a Coinbase-operated [Facilitator](https://github.com/coinbase/open-payments-x402) to verify the signature and extract your address.
3. Settles the payment by calling `transferWithAuthorization` on-chain (the facilitator signs and bundles the transaction).
4. Injects your payer address into the request context.
5. Executes your endpoint (company creation, inference, etc.).
6. Returns the result with an `X-Payment-Receipt` header containing the transaction hash.

**Response (200 OK)**:

```json
{
  "entity_id": "ent_...",
  "trust_address": "pending",
  "runtime_url": "https://ent-....aeqi.ai",
  "owner": "0x...",
  "blueprint": "aeqi",
  "payment_tx": "0x..."
}
```

**Response Header**:

```
X-Payment-Receipt: {
  "txHash": "0x...",
  "payer": "0x...",
  "network": "base"
}
```

## Failure Modes & Debugging

| Status | Error | Meaning | Fix |
|--------|-------|---------|-----|
| 402 | `payment_required` | No `X-Payment` header found. | Construct and sign the authorization (Steps 2–3). |
| 402 | `invalid_payment` | Signature verification failed (bad nonce, expired window, or wrong chain). | Ensure nonce is unique, window is current, and you signed for Base (chainId 8453). |
| 402 | `settlement_failed` | On-chain transfer failed (insufficient balance, contract paused, etc.). | Check your USDC balance and that Base is not congested. |
| 400 | `invalid_name` | Company name is empty or >64 characters. | Trim and validate the name field. |
| 400 | `invalid_blueprint` | Blueprint slug does not exist. | Use `"aeqi"` or query `/api/blueprints` for the list. |
| 500 | `payment_middleware_missing` | Server misconfiguration. | Operator error; contact support. |

## Implementation Notes

### No Account Needed

x402 is account-free. You do not need to:
- Create a user account
- Verify an email
- Wait for approval
- Enter billing information

Your Ethereum private key is your identity. Sign once, execute immediately.

### Custody Model

aeqi never holds your private keys or USDC:

- You retain full control of your Ethereum account.
- You sign the authorization directly (in your application, in a browser, in a CLI tool).
- The facilitator (Coinbase) only signs the on-chain settlement transaction—it cannot steal funds or sign arbitrary transactions for you.
- The payment is immediately visible in the [Base block explorer](https://basescan.org/).

### Replay Protection

Each authorization has:

- A **nonce**: Unique per request; prevents the same signature from being replayed.
- A **validBefore** window: Typically 300 seconds; protects against time-travel attacks.
- A **chain ID**: Bound to Base mainnet (8453); prevents cross-chain replays.

Always generate a fresh nonce and signature for each request.

### Pricing

Company creation: **$19 USDC** (one-time).  
Inference: **Varies** (see [Inference API](/docs/api/inference) for per-model, per-token rates).

Both are final—no contracts, no monthly charges, no hidden fees.
